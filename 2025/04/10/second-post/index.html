SQL注入

原理

SQL 注入是藉由系統內部原本用來存取資料庫的介面，透過注入特殊字元或字串來竄改該介面的原有功能，從而達成攻擊目的。

(這是我在打CTF的簡單範例，CTF題目名稱SSTI1)

解題過程

題目開始就給我題目開始就給我一個介面，輸入文字可以印出。

假設我輸入kurieita按確定。

就會印出kurieita

這其實是一個簡單的 SQL 注入類型的題型，但背後的核心問題與伺服器端模板注入（SSTI）有關。當一個應用程式允許使用者提交資料，且這些資料在未經嚴格驗證或清理的情況下，被直接傳遞給伺服器端的模板引擎進行處理，就可能導致 SSTI 漏洞。
模板引擎的設計目的是動態生成 HTML 等內容，讓網頁能根據不同的資料產生對應的畫面。然而，若使用者提供的輸入未經過濾就被渲染進模板中，攻擊者便可透過特製的輸入來注入惡意程式碼，進而在伺服器上執行非預期的操作。
在這種情況下，為了識別應用程式正在使用哪個模板引擎，我們可以注入特定的有效負載並觀察回應，來測試。
